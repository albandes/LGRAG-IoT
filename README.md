# IoT LGPD Compliance Dataset

Synthetic dataset designed to evaluate AI-based approaches for verifying compliance with the Brazilian General Data Protection Law (LGPD) in Internet of Things (IoT) environments.

## Overview

This dataset simulates data flows generated by IoT devices and their associated privacy metadata. Each record represents a data processing scenario that may or may not comply with LGPD principles.

The dataset was created to support research in:

- AI for privacy compliance
- IoT data governance
- Retrieval-Augmented Generation (RAG)
- Explainable AI for legal reasoning
- Automated compliance verification

## Dataset Characteristics

The dataset contains **500 synthetic IoT data processing scenarios**, each describing:

- device type
- type of personal data
- data subject
- processing purpose
- legal basis
- data sharing behavior
- storage location
- retention policy
- anonymization status
- consent status

Each scenario is labeled with the expected compliance status according to simplified LGPD rules.

## Fields

| Column | Description |
|------|------|
| event_id | Unique identifier of the scenario |
| device_type | Type of IoT device |
| data_type | Type of collected data |
| data_subject | Data subject category |
| purpose | Declared purpose of processing |
| data_storage | Data storage location |
| data_shared | Whether the data is shared with third parties |
| legal_basis | Declared legal basis |
| retention_days | Data retention period |
| anonymized | Whether the data is anonymized |
| consent | Whether consent was obtained |
| compliance_expected | Expected compliance label (1 = compliant, 0 = non-compliant) |
| scenario_description | Short textual description |

## Compliance Label

The label is generated using simplified LGPD-inspired rules combining:

- legal basis
- consent
- data sharing
- anonymization

## Intended Use

This dataset can be used to evaluate:

- LLM-based compliance verification
- Retrieval-Augmented Generation systems
- Explainable AI in legal contexts
- IoT privacy monitoring architectures
